# 광목·패턴지 대여 관리 — 공유 설정 가이드

선생님들이 **같은 데이터를 함께** 보고 입력하려면 두 가지를 합니다.
① 공용 데이터베이스(Supabase) 연결 → ② 웹에 올려서 링크 공유.
전부 **무료**이고, 한 번만 설정하면 됩니다. (예상 소요 10~15분)

설정 전에도 파일을 그냥 더블클릭하면 동작하지만, 그 경우 **각자 자기 컴퓨터에만 저장**됩니다.
화면 상단 배너로 현재 모드를 알려줍니다:
- 🟢 "공용 데이터베이스 연결됨" → 모두가 같은 데이터
- 🟠 "이 브라우저에만 저장 중" → 아직 공유 안 됨 (아래 1단계 필요)

---

## 1단계 — Supabase 공용 데이터베이스 만들기 (무료)

1. https://supabase.com 접속 → **Start your project** → 깃허브/구글 계정으로 가입
2. **New project** 클릭
   - Name: 아무거나 (예: `papertextile`)
   - Database Password: 아무거나 정하고 메모해두기 (나중에 안 쓰여도 됨)
   - Region: `Northeast Asia (Seoul)` 권장
   - **Create new project** (1~2분 기다림)

3. 왼쪽 메뉴 **SQL Editor** → **New query** → 아래를 통째로 붙여넣고 **Run** ▶

```sql
-- 학생 명단
create table public.students (
  id   text primary key,
  name text not null,
  sid  text default ''
);

-- 대여 기록 (한 건 = 한 행)
create table public.rentals (
  id         bigint generated always as identity primary key,
  student_id text not null references public.students(id) on delete cascade,
  rented_on  date not null,
  muslin     int  not null default 0,
  pattern    int  not null default 0,
  created_at timestamptz default now()
);

-- 누구나(앱 사용자) 읽고 쓸 수 있도록 허용
alter table public.students enable row level security;
alter table public.rentals  enable row level security;

create policy "anyone can read students"  on public.students for select using (true);
create policy "anyone can write students" on public.students for all    using (true) with check (true);
create policy "anyone can read rentals"   on public.rentals  for select using (true);
create policy "anyone can write rentals"  on public.rentals  for all    using (true) with check (true);

-- 실시간 동기화 켜기 (다른 선생님 입력이 자동 반영됨)
alter publication supabase_realtime add table public.students;
alter publication supabase_realtime add table public.rentals;
```

"Success. No rows returned" 이 뜨면 성공입니다.

4. 왼쪽 메뉴 **Project Settings**(톱니바퀴) → **API** 에서 두 값을 복사:
   - **Project URL** (예: `https://abcdefgh.supabase.co`)
   - **Project API keys → `anon` `public`** (길게 시작하는 `eyJ...` 키)

---

## 2단계 — 앱에 키 붙여넣기

`papertextile.html` 파일을 메모장/편집기로 열고, 맨 위쪽의 이 두 줄을 찾아 붙여넣습니다:

```html
<script>
  window.SUPABASE_URL = "여기에 Project URL 붙여넣기";
  window.SUPABASE_ANON_KEY = "여기에 anon public 키 붙여넣기";
</script>
```

저장 후 파일을 더블클릭해 열어, 상단 배너가 🟢 **"공용 데이터베이스 연결됨"** 으로 바뀌면 완료입니다.
학생을 한 명 추가해보고, 다른 컴퓨터(또는 시크릿창)에서 같은 파일을 열어 같은 학생이 보이면 공유 성공.

> ⚠️ `anon public` 키는 웹에 올려도 되는 공개용 키입니다. `service_role` 키는 절대 쓰지 마세요.

---

## 3단계 — 웹에 올려서 링크로 공유 (무료, 택1)

### 방법 A) Netlify Drop — 가장 쉬움 (드래그 한 번)
1. https://app.netlify.com/drop 접속
2. `papertextile.html` 파일을 화면에 **끌어다 놓기**
   - (이름을 `index.html`로 바꿔서 올리면 주소가 더 깔끔해집니다)
3. 몇 초 뒤 `https://랜덤이름.netlify.app` 링크가 생깁니다 → 선생님들께 공유 끝

### 방법 B) GitHub Pages
1. github.com 가입 → 새 저장소(repository) 생성 (Public)
2. `papertextile.html`을 `index.html`로 올리기(upload)
3. Settings → Pages → Source를 `main` 브랜치로 → 저장
4. 잠시 후 `https://아이디.github.io/저장소이름/` 링크 발급

둘 다 무료이고, 한 번 올리면 그 링크로 모두가 같은 공용 데이터를 사용합니다.
앱을 수정하면 같은 방법으로 다시 올리면(또는 파일 교체) 됩니다.

## 백업
앱의 **CSV 내보내기** 버튼은 현황 CSV와 함께 전체 백업(JSON)도 내려줍니다.
가끔 눌러 JSON을 보관해두면, 사고 시 **불러오기**로 복원할 수 있습니다.
